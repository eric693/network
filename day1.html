Section 01

網路的定義與本質

電腦網路（Computer Network）是指兩台或兩台以上的電腦設備， 透過某種傳輸媒介（有線或無線）相互連結，並依照共同認可的 通訊協定（Protocol）進行資料交換的系統。

這個定義包含三個核心要素：「設備（端點）」、「媒介（通道）」與「協定（規則）」。 缺少任何一個，通訊都無法發生。就像人類的語言：說話者是設備， 空氣是媒介，語法規則就是協定。

核心概念

網路的本質是資源共享與訊息傳遞。 資源共享讓多台設備共用印表機、檔案或運算能力；訊息傳遞則讓人與人、 機器與機器之間能夠即時溝通。現代網際網路是這兩者的極致體現。

為什麼需要網路？

在網路出現之前，資料必須靠實體媒體（磁碟、磁帶）人工傳遞， 稱為「鞋底網路（Sneakernet）」。這種方式速度慢、成本高且難以擴展。 電腦網路解決了以下問題：

需求 說明 現代應用範例
資源共享 多人共用同一台印表機、儲存空間或應用程式 NAS 儲存、雲端硬碟
通訊 在不同地理位置的使用者之間傳遞訊息 電子郵件、即時通訊
分散運算 將運算任務拆分給多台機器平行處理 Hadoop、分散式資料庫
冗餘備援 某節點故障時，訊息可繞路繼續傳遞 CDN、多路徑路由
集中管理 透過網路統一設定與監控多台設備 DevOps、IoT 管理平台
Section 02

歷史演進

了解網路的歷史，能幫助我們理解為何今日的技術選擇如此設計。 許多協定的「怪異之處」，往往源自幾十年前的工程決策。

1957
冷戰催生網路研究

蘇聯發射史普尼克衛星，美國國防部成立 ARPA（高等研究計劃局）， 開始研究具備核戰生存能力的去中心化通訊網路。

1969
ARPANET 誕生

世界上第一個封包交換網路上線，連接加州大學洛杉磯分校、 史丹佛研究院、加大聖塔芭芭拉分校與猶他大學等四個節點。 第一條訊息是「lo」——傳輸「login」時系統即崩潰。

1974
TCP/IP 概念提出

Vint Cerf 與 Bob Kahn 發表論文，提出傳輸控制協定概念， 奠定現代網際網路的通訊基礎。兩人後來被稱為「網際網路之父」。

1983
DNS 與 TCP/IP 標準化

ARPANET 正式切換至 TCP/IP 協定族，同年 DNS 系統提出， 取代人工維護的 HOSTS.TXT 主機名稱對應檔。

1991
全球資訊網（WWW）誕生

Tim Berners-Lee 在歐洲核子研究組織（CERN）發明 HTTP 與 HTML， 讓網路從學術工具蛻變為大眾媒介。這是網際網路史上最重要的轉折點之一。

1998
IPv6 規格發布

面對 IPv4 位址即將耗盡的危機，IETF 發布 IPv6 標準， 將位址空間從 32 位元擴展至 128 位元，理論上可提供 3.4×10³⁸ 個位址。

2010s
行動網路與物聯網時代

4G LTE 普及、智慧手機成為主要上網工具；IoT 設備數量超越人口數量， 推動邊緣運算與 5G 技術的快速發展。

Section 03

網路分類

網路可以從多個維度分類：依地理範圍、依擁有權、依傳輸技術。 最常見的分類方式是依地理覆蓋範圍：

類型 全名 覆蓋範圍 典型速度 範例
PAN 個人區域網路 1–10 公尺 1–480 Mbps 藍牙耳機、USB 連線
LAN 區域網路 建築物內 100 Mbps–10 Gbps 辦公室乙太網路
MAN 都會區域網路 城市範圍 10 Mbps–1 Gbps 有線電視網路、城市 Wi-Fi
WAN 廣域網路 國家/全球 不定 網際網路、企業 VPN
WLAN 無線區域網路 建築物內 54 Mbps–9.6 Gbps 家用 Wi-Fi（IEEE 802.11）

依擁有權分類

公用網路（Public Network）由電信業者或政府維運， 任何人只要付費即可使用，網際網路是最典型的例子。 私有網路（Private Network）則由特定組織擁有與管理， 通常僅供內部員工使用，例如企業內網或政府機關的隔離網路。 虛擬私有網路（VPN）則是在公用網路上建立加密通道， 模擬私有網路的安全性，後續 Day 15 會深入討論。

電路交換 vs. 封包交換

這是網路設計中最根本的二元選擇。電路交換（Circuit Switching）在 通訊開始前先建立一條專用線路，通話期間獨佔頻寬，傳統電話系統即採用此模式。 優點是延遲穩定、品質有保障；缺點是資源利用率低，線路閒置時頻寬白白浪費。

封包交換（Packet Switching）則將資料切成小塊（封包）， 每個封包獨立選擇路徑傳輸，到達目的地後再重組。網際網路即採用此架構。 優點是頻寬共享效率高；缺點是延遲可能不穩定（抖動）。

電路交換 vs. 封包交換示意
電路交換（Circuit Switching）
  A ─────────────────────────────── B
    ← 專屬線路，通話中其他人無法使用 →

封包交換（Packet Switching）
  A → [P1][P2][P3] → 路由器 → [P2][P1][P3] → B
           ↑ 每個封包可走不同路徑，到達後重組 ↑
Section 04

網路拓樸結構

拓樸（Topology）描述網路中節點的連接方式， 分為實體拓樸（電纜如何鋪設）與 邏輯拓樸（資料如何流通）。 兩者可以不同：例如實體星狀拓樸在邏輯上可能運作為匯流排拓樸。

Bus（匯流排）
A—B—C—D—E |___________|

所有節點共用一條主幹線。安裝簡單成本低，但主幹線斷掉整個網路失效， 碰撞問題嚴重，現代已少用。

Ring（環狀）
A / \ E B | | D C \ /

資料單向繞環傳遞，使用令牌（Token）控制發送權。 IEEE 802.5 Token Ring，延遲可預測，但單點故障影響全環。

Star（星狀）
A B C \|/ H /|\ D E F

所有節點連到中央 Hub/Switch。管理容易、單點故障隔離， 是現代乙太網路（LAN）最常見的實體拓樸。

Mesh（網狀）
A—B—C |\ |\ | | \| \| D—E—F

每個節點都與其他多個節點直接相連。 容錯性極高，常用於骨幹網路與軍事通訊，但佈線成本昂貴。

Tree（樹狀）
Root / \ A B / \ / \ C D E F

星狀拓樸的延伸，透過層級結構擴展規模。 企業園區網路常見，易於擴充，但根節點故障影響全樹。

Hybrid（混合）
Star + Mesh [核心]Mesh / | \ S S S ← 各分支為 Star

結合多種拓樸優點。現代大型企業與雲端資料中心幾乎都採用混合拓樸設計。

Section 05

基本硬體元件

了解網路設備的功能是理解訊號如何流動的基礎。 不同設備運作在 OSI 模型的不同層（下一課深入討論），功能截然不同。

設備 OSI 層 功能 智慧程度
網路卡（NIC） Layer 1–2 讓設備與網路物理連接，負責訊號轉換與 MAC 位址識別 固定
集線器（Hub） Layer 1 將訊號廣播至所有連接埠，不區分目的地，效率低 無
交換器（Switch） Layer 2 學習 MAC 位址，將訊框只送往正確連接埠，避免廣播浪費 中等
路由器（Router） Layer 3 根據 IP 位址在不同網路之間轉送封包，是網際網路的核心設備 高
閘道器（Gateway） Layer 4–7 在使用不同協定的網路之間進行轉換，例如 VoIP 閘道器 最高
中繼器（Repeater） Layer 1 放大訊號，解決長距離傳輸的訊號衰減問題 無
橋接器（Bridge） Layer 2 連接兩個相同類型的區段，過濾並轉送訊框，為早期交換器前身 低

常見混淆

家用「無線路由器」其實是 Router + Switch + Access Point 三合一設備。 它既扮演連接 ISP 的路由角色，也扮演連接家中多台設備的交換器角色， 同時提供 Wi-Fi 無線接入點功能。

Section 06

資料傳輸基礎

頻寬、吞吐量與延遲

這三個指標是衡量網路效能最重要的維度，日常上常被混用但意義截然不同：

指標 定義 單位 類比
頻寬（Bandwidth） 通道理論上可傳輸的最大資料速率 bps / Mbps / Gbps 水管的管徑大小
吞吐量（Throughput） 實際上單位時間傳輸的有效資料量 bps 實際流過水管的水量
延遲（Latency） 封包從來源到目的地所需的時間 ms 水從一端流到另一端的時間
抖動（Jitter） 延遲的變化量，對即時應用影響大 ms 水流速度的不規則波動

重要觀念

高頻寬不等於低延遲。下載大文件需要高頻寬；線上遊戲或視訊通話更在意低延遲。 頻寬好比高速公路的車道數，延遲好比城市之間的距離，兩者是獨立的維度。

傳輸模式

資料傳輸有三種基本方向性模式： 單工（Simplex）只能單向傳輸，例如廣播電視； 半雙工（Half-Duplex）可雙向但不同時，例如對講機； 全雙工（Full-Duplex）可同時雙向傳輸，例如電話和現代乙太網路。

數位訊號與類比訊號

類比訊號是連續波形，振幅、頻率隨時間連續變化，傳統電話線使用類比訊號。 數位訊號只有離散狀態（0 與 1），現代網路幾乎全面使用數位訊號。 調變（Modulation）是將數位訊號轉換為適合傳輸媒介的波形的技術， 例如 ADSL 使用 DMT 調變技術讓數位資料在傳統電話線上傳輸。

位元與位元組

網路速度通常以位元（bit）為單位（小寫 b）， 而檔案大小通常以位元組（Byte）為單位（大寫 B）， 1 Byte = 8 bits。因此 100 Mbps 的網路速度， 理論上最大下載速度是 100 ÷ 8 = 12.5 MB/s。 許多人誤以為 100 Mbps 可以每秒下載 100 MB，這是最常見的單位混淆。

Section 07

通訊模式：Client-Server vs. P2P

現代網路應用建立在兩種基本架構之上，理解它們的差異是後續學習 HTTP、 DNS、檔案傳輸等應用層協定的前提。

主從式架構（Client-Server）

網路中有明確的服務提供者（Server）與 服務使用者（Client）之分。 伺服器持續運行，等待用戶端的連線請求；用戶端主動發起連線， 提出請求並等待回應。Web 瀏覽（HTTP）、電子郵件（SMTP/IMAP）、 DNS 查詢都採用此模式。

Client-Server 通訊流程
Client                          Server
  │                                │
  │──── TCP SYN ──────────────────>│
  │<─── TCP SYN-ACK ───────────────│
  │──── TCP ACK ──────────────────>│  (連線建立)
  │                                │
  │──── HTTP GET /index.html ─────>│  (請求)
  │<─── HTTP 200 OK + HTML ────────│  (回應)
  │                                │
  │──── TCP FIN ──────────────────>│  (關閉連線)

對等式網路（Peer-to-Peer, P2P）

每個節點（Peer）既是客戶端也是伺服器，資源分散在所有參與節點上。 BitTorrent 是最知名的 P2P 應用：下載者同時也在上傳， 節點越多下載速度反而越快。P2P 的優勢是去中心化、高可用性， 缺點是版權問題難以管控，且難以保證服務品質。

現代應用常見混合架構：BitTorrent 使用中央 Tracker 伺服器來協調節點， 但實際資料傳輸是 P2P 進行；區塊鏈也是 P2P 概念的延伸。

Section 08

標準化組織

網路世界能互通運作，靠的是各組織制定並推行的統一標準。 理解這些組織能幫助你找到最權威的技術文件來源。

組織 全名 負責範疇 代表成果
IETF 網際網路工程任務組 網際網路協定標準（RFC 文件） TCP、UDP、HTTP、DNS
IEEE 電機電子工程師學會 有線與無線區域網路實體與資料鏈結層標準 802.3 乙太網路、802.11 Wi-Fi
ISO 國際標準化組織 廣泛的技術標準，包含 OSI 模型定義 OSI 參考模型（ISO 7498）
ICANN 網際網路名稱與數位位址分配機構 管理全球網域名稱與 IP 位址分配 DNS 根伺服器管理、TLD 授權
ITU 國際電信聯盟（聯合國機構） 電信與廣播標準，頻譜分配 G.711 語音編碼、H.264 視訊
W3C 全球資訊網協會 Web 相關技術標準 HTML、CSS、WebSocket

RFC 文件

RFC（Request for Comments）是 IETF 發布的技術標準文件。 幾乎所有網際網路核心協定都有對應的 RFC，例如 TCP 是 RFC 793、 HTTP/1.1 是 RFC 7230–7235。這些文件可在 rfc-editor.org 免費查閱， 是最權威的協定參考資料。

Day 1 重點回顧

今天你學到了什麼

  • 網路的三要素：端點設備、傳輸媒介、通訊協定，缺一不可
  • 網路依範圍分為 PAN、LAN、MAN、WAN；依技術分為電路交換與封包交換
  • ARPANET 1969 年誕生，TCP/IP 1974 年提出，WWW 1991 年開放，奠定現代網路
  • 六大拓樸：Bus、Ring、Star（最常用）、Mesh、Tree、Hybrid
  • Hub 廣播、Switch 學習 MAC 位址、Router 轉送 IP 封包，功能層次不同
  • 頻寬是上限，吞吐量是實際值，延遲是時間，三者獨立衡量
  • Client-Server 是主流架構；P2P 去中心化，BitTorrent 為代表
  • IETF 的 RFC 是最權威的協定規格文件來源
